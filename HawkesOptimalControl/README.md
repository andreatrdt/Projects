# Hawkes Optimal Control

This project combines three layers for high-frequency market-making research on LOBSTER-style limit-order-book data.

## Architecture

```text
LOBSTER message/orderbook data
        |
        v
Event cleaning and Hawkes calibration
        |
        v
Hawkes intensity state: lambda_plus, lambda_minus
        |
        +----------------------+
        |                      |
        v                      v
HJB optimal quoting       ML price interval forecasting
        |                      |
        v                      v
Bid/ask quotes, PnL,      Short-horizon predictive bands
inventory path            for future mid-price
```

The project is no longer only a Hawkes/HJB thesis implementation. The ML interval layer is designed as a third pillar: it estimates short-horizon uncertainty around the next mid-price conditional on the Hawkes/control state.

## Main components

### 1. Hawkes order-flow model

The Hawkes layer models aggressive buy/sell order arrivals using calibrated self- and cross-exciting intensities. The reduced 2D setup uses the execution-side components:

- `E_a`: aggressive buy / ask-side execution flow.
- `E_b`: aggressive sell / bid-side execution flow.

The fitted Hawkes state produces the intensity paths `lambda_plus` and `lambda_minus`, which are then used by the control and ML layers.

### 2. HJB optimal control layer

The control layer solves a finite-difference HJB problem for a risk-averse market maker. The policy maps the state

```text
(t, S, q, lambda_plus, lambda_minus)
```

into optimal bid and ask offsets. The backtester then produces quoted prices, inventory, fills and PnL diagnostics.

### 3. ML next-price interval layer

The ML layer lives in:

```text
code/ml_interval_forecaster.py
```

It predicts conditional quantiles of the short-horizon future mid-price change:

```text
Y_t = S_{t+h} - S_t
```

The predicted quantiles are converted into price bands by adding the current mid-price:

```text
S_t + q_0.05(Y_t), S_t + q_0.50(Y_t), S_t + q_0.95(Y_t)
```

This is a probabilistic forecast, not a pure point forecast. The useful output is the interval and its diagnostics.

## ML features

The ML layer uses features generated by the Hawkes/HJB pipeline, including:

- Hawkes intensities: `lambda_plus`, `lambda_minus`.
- Flow imbalance: `(lambda_plus - lambda_minus) / (lambda_plus + lambda_minus)`.
- Current inventory `q`.
- Bid/ask quotes and quoted spread.
- Recent mid-price changes and realized volatility.
- Fill probabilities and fill imbalance.

All splits are chronological. Random train/test splitting is intentionally avoided to reduce time-series leakage.

## Models

The interval forecaster supports:

- `gbrt_quantile`: main baseline, one gradient-boosting quantile regressor per quantile.
- `rf_quantile`: random-forest empirical intervals from tree-level predictions.
- `mlp_quantile`: exploratory neural-network variant.

The recommended baseline is `gbrt_quantile`.

## Diagnostics

The key out-of-sample diagnostics are:

- empirical coverage,
- coverage error versus nominal coverage,
- average interval width,
- pinball loss,
- Winkler interval score,
- median MAE,
- directional accuracy.

For a 5%-95% interval, the nominal coverage is 90%. If empirical coverage is too low, the intervals are too narrow. If it is too high, the intervals may be too conservative.
